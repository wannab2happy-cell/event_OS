# 배포 후 테스트 체크리스트

## 📋 전체 테스트 체크리스트

### 1. 기본 접속 테스트

- [ ] **홈페이지 접속**
  - 배포된 URL로 접속 (`https://[your-domain].vercel.app`)
  - 페이지가 정상적으로 로드되는지 확인
  - 에러 메시지가 없는지 확인

- [ ] **이벤트 랜딩 페이지 접속**
  - `https://[your-domain].vercel.app/[eventId]` 접속
  - 이벤트 정보가 정상적으로 표시되는지 확인
  - 브랜딩 이미지(로고, KV 이미지)가 정상적으로 로드되는지 확인

### 2. Admin 기능 테스트

- [ ] **Admin 로그인**
  - `/admin/login` 페이지 접속
  - Admin 계정으로 로그인 시도
  - 로그인 성공 후 대시보드로 리다이렉트되는지 확인

- [ ] **Admin 대시보드**
  - `/admin/dashboard` 접속
  - 통계 카드가 정상적으로 표시되는지 확인
  - 최근 참가자 목록이 표시되는지 확인

- [ ] **참가자 목록 조회**
  - `/admin/events/[eventId]/participants` 접속
  - 참가자 목록이 정상적으로 표시되는지 확인
  - 상태별 필터링이 작동하는지 확인

- [ ] **참가자 상세/편집**
  - 참가자 목록에서 특정 참가자 클릭
  - 참가자 정보가 정상적으로 표시되는지 확인
  - 확정 정보 입력 폼이 작동하는지 확인

- [ ] **브랜딩 설정**
  - `/admin/events/[eventId]/branding` 접속
  - 이미지 업로드가 정상적으로 작동하는지 확인
  - 업로드된 이미지가 정상적으로 표시되는지 확인

### 3. 참가자 기능 테스트

- [ ] **참가자 로그인 (Magic Link)**
  - 이벤트 랜딩 페이지에서 "등록 정보 입력하기" 클릭
  - 이메일 입력 후 Magic Link 요청
  - 이메일로 Magic Link 수신 확인
  - Magic Link 클릭 후 로그인 성공 확인

- [ ] **참가자 등록 폼**
  - `/[eventId]/register?step=basic-info` 접속
  - 기본 정보 입력 폼이 정상적으로 표시되는지 확인
  - 정보 입력 후 저장 버튼 클릭
  - 저장 성공 메시지 확인

- [ ] **여권 정보 입력**
  - `/[eventId]/register?step=passport` 접속
  - 여권 정보 입력 후 저장
  - 저장 성공 확인

- [ ] **항공 정보 입력**
  - `/[eventId]/register?step=flight` 접속
  - 항공 정보 입력 후 저장
  - 저장 성공 확인

- [ ] **호텔 정보 입력**
  - `/[eventId]/register?step=hotel` 접속
  - 호텔 정보 입력 후 저장
  - 저장 성공 및 등록 완료 확인

- [ ] **QR PASS 페이지**
  - `/[eventId]/qr-pass` 접속
  - QR 코드가 정상적으로 표시되는지 확인
  - 참가자 정보가 정상적으로 표시되는지 확인
  - 항공/호텔 확정 정보가 표시되는지 확인 (확정된 경우)

### 4. 이메일 발송 테스트

- [ ] **확정 메일 발송**
  - Admin에서 참가자의 항공 또는 호텔 확정 정보 입력
  - 확정 상태 변경 (`is_travel_confirmed` 또는 `is_hotel_confirmed`)
  - 참가자 이메일로 확정 메일이 발송되는지 확인
  - 이메일 내용이 정상적으로 표시되는지 확인
  - 이메일의 링크가 정상적으로 작동하는지 확인

### 5. 보안 테스트

- [ ] **RLS 정책 테스트**
  - 참가자 A로 로그인
  - 참가자 A의 데이터만 조회/수정 가능한지 확인
  - 참가자 B의 데이터에 접근 불가능한지 확인

- [ ] **Admin 권한 테스트**
  - Admin으로 로그인
  - 모든 참가자 데이터에 접근 가능한지 확인
  - 참가자 정보 수정이 정상적으로 작동하는지 확인

### 6. 성능 테스트

- [ ] **페이지 로딩 속도**
  - 각 주요 페이지의 로딩 속도 확인
  - 이미지 로딩 속도 확인
  - 3초 이내 로딩 완료 확인 (권장)

- [ ] **반응형 디자인**
  - 모바일 화면에서 정상적으로 표시되는지 확인
  - 태블릿 화면에서 정상적으로 표시되는지 확인
  - 데스크톱 화면에서 정상적으로 표시되는지 확인

### 7. 에러 모니터링

- [ ] **Vercel 로그 확인**
  - Vercel 대시보드 → Functions → Logs
  - 에러 로그가 없는지 확인
  - 경고 로그 확인 및 필요시 수정

- [ ] **브라우저 콘솔 확인**
  - 브라우저 개발자 도구 → Console
  - JavaScript 에러가 없는지 확인
  - 네트워크 에러가 없는지 확인

### 8. 데이터 무결성 테스트

- [ ] **데이터 저장 확인**
  - 참가자 정보 입력 후 저장
  - Supabase에서 데이터가 정상적으로 저장되었는지 확인
  - 데이터 형식이 올바른지 확인

- [ ] **데이터 조회 확인**
  - 저장된 데이터가 정상적으로 조회되는지 확인
  - 필터링 및 정렬이 정상적으로 작동하는지 확인

## 🐛 문제 발생 시 체크리스트

### 빌드 실패 시

1. Vercel 빌드 로그 확인
2. 환경 변수 누락 확인
3. 로컬에서 `npm run build` 실행하여 오류 확인

### 로그인 실패 시

1. Supabase Redirect URL 설정 확인
2. 환경 변수 (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`) 확인
3. Supabase Authentication 설정 확인

### 이메일 발송 실패 시

1. `RESEND_API_KEY` 환경 변수 확인
2. `RESEND_DOMAIN` 환경 변수 확인
3. Resend 대시보드에서 발송 로그 확인

### 데이터 조회 실패 시

1. RLS 정책 설정 확인
2. `SUPABASE_SERVICE_ROLE_KEY` 환경 변수 확인 (Admin 기능)
3. Supabase 연결 상태 확인
